Django upgrade (1.4 to 1.9) in a 100+ app project

Romain Garrigues (@rgarrigues)

Context @ iwoca

  • Aout 2014: end of life for Django 1.4 LTS
  • Quite big monolithic Django project (100+ custom apps)
  • Continuous deployment (~10 releases/day)
  • Continuous deployment (~10 releases/day)
  • 20 developers full time

Problem to solve

  • Massive upgrade (1.4 to 1.8)
  • Upgrade current code while 1.4 code still produced

Solution

  • One version a time (1.5, 1.6, 1.7, 1.8, 1.9)
  • Read Django Release
  • Create env with new Django version
  • Run tests
  • Fix each issue
